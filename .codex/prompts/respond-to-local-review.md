# Respond To Local Review

Read `docs/reviews/pr-<PR_NUMBER>.md`, the PR diff, and the original Codex handoff task.

Address blocking issues first. Keep suggestions separate from required changes. Do not broaden scope unless Claude Code or the human explicitly requests it.

Do not open a new PR for review fixes. Push the fix to the same feature branch (updating the existing PR) and record the follow-up as a comment on that PR.
