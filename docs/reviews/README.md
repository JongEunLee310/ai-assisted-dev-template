# Reviews

## Directory Purpose

This directory stores durable local PR review records written by Claude Code, per `docs/harness/local-review-policy.md`.

## File Naming Rules

Use `pr-<number>.md`, one file per PR.

## Required Sections

Every review record uses this fixed section order and headers:

1. `## Review Summary`
2. `## Blocking`
3. `## Suggestions`
4. `## Questions`
5. `## CI Result`
6. `## Documentation Impact`
7. `## Final Recommendation`

Use the headers even when a section is empty (state "None").
